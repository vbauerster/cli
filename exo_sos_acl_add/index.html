exo sos acl add

Add ACL(s) to object(s)
exo sos acl add <bucket name> <object name> [object name] ... [flags]

Options

--private, -p - Canned ACL private
--public-read, -r - Canned ACL public read
--public-read-write, -w - Canned ACL public read and write
--authenticated-read - Canned ACL authenticated read
--bucket-owner-read - Canned ACL bucket owner read
--bucket-owner-full-control, -f - Canned ACL bucket owner full control
--read - Manual acl edit grant read e.g(value, value, …)
--write - Manual acl edit grant write e.g(value, value, …)
--read-acp - Manual acl edit grant acp read e.g(value, value, …)
--write-acp - Manual acl edit grant acp write e.g(value, value, …)
--full-control - Manual acl edit grant full control e.g(value, value, …)
--help, -h - help for add

Options inherited from parent commands

--config, -C - Specify an alternate config file [env EXOSCALE_CONFIG]
--output-format, -O - Output format, can be table|json|text
--output-template - Template to use if output format is "template"
--use-account, -A - Account to use in config file [env EXOSCALE_ACCOUNT]
